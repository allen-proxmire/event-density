# ED-SIM v1 Reproducibility Suite

## Purpose

ED-SIM v1 is the canonical simulation engine for the Event Density architectural ontology. It implements the PDE system of the Rigour Paper (Appendix C), computes the sixteen invariant families of the Numerical Atlas, and produces the ED Consistency Certificate that summarizes the architecture's structural integrity across all tested parameter regimes.

This reproducibility suite guarantees that every invariant, every figure, and every certificate in the ED-SIM v1 pipeline can be regenerated from scratch on any machine with a standard Python environment. No pre-computed data is required. No manual intervention is needed. The suite runs end-to-end and produces a machine-verifiable output that matches the published results.

The guarantee is threefold:

1. **Bitwise reproducibility** within the same environment (OS, Python version, library versions, hardware).
2. **Statistical reproducibility** across environments: all invariant verdicts (INVARIANT / WEAKLY INVARIANT / NOT INVARIANT) and the final certificate verdict (PASS / PARTIAL / FAIL) are identical.
3. **Structural reproducibility**: the directory tree, file formats, metadata schemas, and figure specifications are invariant across all environments.

## Quick Start

```bash
git clone <repo-url> && cd "ED Simulation"
pip install numpy scipy matplotlib
python reproducibility/run_all.py
# Outputs appear under output/
# Certificate: output/atlas/ED_Consistency_Certificate.txt
```

## Troubleshooting

### Missing dependencies

```bash
python reproducibility/checks/check_environment.py
```
This prints which packages are installed, which are missing, and which are optional. Install missing required packages before running the pipeline.

### Missing runs

If the regime volume experiment has not been run:
```bash
python experiments/regime_volume_3d.py
```
Then re-run the reproducibility suite. The suite will skip invariants for which no data exists and report them as MISSING in the certificate.

### Partial invariants

If some invariant scripts fail (e.g., due to insufficient data), the suite continues with the remaining invariants. The global atlas report and certificate reflect the partial coverage. Re-run the failing script individually:
```bash
python experiments/invariant_<name>.py
```

### Regenerating a single invariant

```bash
python experiments/invariant_low_mode_collapse.py   # example
python reproducibility/validation/validate_outputs.py
```

## Versioning

This suite tracks ED-SIM v1.0.0. All output files contain version metadata. The validation suite checks version compatibility before comparing outputs.
